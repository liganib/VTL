Setting up VTL RESTful services

Deploying the RESTful services web application

To deploy the application, acquire the war package either by Maven or from the GitHub releases page.

To download the package with Maven into Tomcat deployment folder, use:

$ mvn dependency:copy -Dmdep.stripVersion=true \
        -Dartifact=it.bancaditalia.oss.vtl:vtl-rest:1.0.7:war \ 
        -DoutputDirectory=/path/to/tomcat/deploy

After the package has been downloaded, Tomcat should unpack and start it automatically.

Tomcat usually deploys applications under a path that starts with the war file name, so, if the file was download by Maven, the entry point for the services should be in this case /vtl-rest/.

System requirements

The requirements of VTL RESTful services are:

  • Java 8+
  • Web Container compatible with Servlet 3.1
  • Internet connection (optional)

Please note that Jakarta EE technology is not required, although the services application could be run in any of the compatible containers.

Particular configurations

Some of the VTL engine features require an internet connection.

If you are behind a proxy server, you may need additional configuration to enable these features.

CSV Environment

Please note that CSV Environment accesses files located on the container host.

Although it fails if the requested file is not in a CSV format, it may pose a security risk.

If this would be the case to a production environment, it can be disabled by setting the
Java system property.

This property must be set to a sequence of class names, separated by commas, which must implement the Environment interface.

Available implementation classes are found in the Java package it.bancaditalia.oss.vtl.impl.environment.

NOTE: In order for the engine to work correctly, at least one of the chosen classes must also implement the Workspace interface.

SDMX Environment

To enable the use of SDMX environment by the web services, you must follow this document to setup a properties file, which should contain your proxy settings, and then link it with an environment variable.

Choose a VTL metadata repository implementation

This Java system property allows you to choose which implementation class you want to use as a metadata repository.

There are three available implementations, described here.

The default implementation is the In-Memory repository. To choose a different one, you can add a system property in your container configuration.

It must point to a class that implements the MetadataRepository interface, such as:

  • it.bancaditalia.oss.vtl.impl.domains.CSVMetadataRepository
  • it.bancaditalia.oss.vtl.impl.domains.SDMXMetadataRepository

If you want to use the SDMX Metadata Repository, you might also need to set standard Java proxy properties.
